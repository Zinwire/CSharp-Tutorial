# Переменные и типы данных
## Немного теории
Переменные сами из себя представляют контейнер, в котором хранят что-либо, будь то число, строка, или может, символ?<br>
В C# необходимо указывать тип данных переменной, например `int`. Тип данных указывает на то, насколько тип данных будет много занимать места в оперативной памяти, а также то, что он будет хранить.<br>
При разработке маленьких проектов разница между очень маленьким типом данных и очень большим в использовании будет незаметна, но при разработке больших систем, придётся уже думать перед тем, как писать.<br>
Так же хотелось бы отметить, что при переполнении переменной, вы получите ошибку.
Создание переменной выглядит так:
```csharp
//<тип данных> <название> = <значение>;
//Для примера:
int num = 69;
byte H = 52;
string k = "Да, это строка";
```
## Какие типы данных вообще есть?
Предлагаю составить таблицу, приводя примеры использования, и описывать типы.

Целочисленные типы данных - типы данных, которые могут хранить целое число, но не могут хранить дробную его часть, округляясь до меньшего при присутствии дроби<br>
1. `byte` - самый маленький из всех. Весит 1 байт (Логично). Переменная может хранить значение от 0 до 255 . Пример: `byte i = 255`. Но `byte o = 256` выдаст ошибку.
2. `short` - весит 2 байта, вмещает от -32 768 до 32 767. `short Short = -32768`.
3. `int` - самый популярный. Весит 4 байта, и хранит от -2147483648 до 2147483647. `int Int = 67`
4. `long` - весит 8 байт и хранит от -9квнтлн до 9квнтлн (очень много). `long Very_Long = 39424234`

Дробные типы данных - типы данных, способные хранить дробную часть числа. 
1. `float` - весит 4 байта, хранит до 6 цифр после запятой, требует *f* на конце для обозначения. `float Ft = 5,89894f`
2. `double` - весит 8 байт, хранит до 15 цифр после запятой. Не требует *f* или *d*. `double trouble = 5.555555555555555`
3. `decimal` - весит 16 байт, 29 цифр после запятой, наценел на высокую точность. `decimal animal = 6.66`

Остальные типы данных - уникальные типы.
1. `bool` - хранит только два возможных значения, а именно `true`(да) или `false`(нет). `bool Booling = true`(значение без ковычек)
2. `char` - хранит один текстовый символ. Объявляется значение в `'`. `char space = ' '`
3. `string` - хранит текст длиннее двух букв. `string WriteLine = "Антон Чигур"`
4. `var` - автоматически подстраивается основываясь на вводе. `var i = 3`-число будет `long`, `var Pi = 3,14`-число будет double, `var Gh = true`-будет `bool`

## Что с этими данными можно делать?
Эти данные можно использовать например, для ввода в функцию
```csharp
string Word = "Текст";
Console.WriteLine(Word); //Выведет: "Текст"
```

Так же числа можно делить/умножать/складывать/вычитать
делается это так:
```csharp
//<тип данных> <имя> = <число 1> <знак> <число 2>
int g = 4; int h = 2; int f; //Переменные так же можно просто объявить на будущее
f = g / h // `/` отвечает за деление, так же * умножает, + складывает, - вычитает
```
Числа тоже можно выводить
```csharp
int f = 5, h = 3, y; //Так объявлять тоже можно, ведь все они int
y = 5 / 3; //будет 1, т.к. полностью поделилось только 1 раз. Будь числа дробными, было бы 1.66...
Console.Write(y); //Выведет "1"
```
Так же есть удобные операторы для быстрого вычисления без лишних данных
```csharp
float y = 5f, j = 3f;
y *= j; //Сначала пишем действие, потом равно, чтобы кратко записать
Console.Write(y); //>>>15
```

Хотелось бы отметить, тот же `char` или `string` складывать нельзя.<br>
Или можно?
```csharp
char h = 'h', i = 'i';
h += i; //опа, ошибка, хранить больше одного знака нельзя. Но давайте представим что это теперь string и получилось "hi"
Console.Write(h); // Будь это string, вывело бы "hi"
```
