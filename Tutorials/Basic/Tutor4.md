# U-Типы данных и конвертация
## U-типы данных
U-типы данных представляют из себя те же типы, но без отрицательных значений, например `uint`.<br>
Зачем это нужно если есть базовые типы данных? Ну, место под отрицательные значения уходит на положительные.<br>
Рассмотрим `short` и `ushort`.<br>
Значения, которые может хранить `short`, это от -32768 до 32767 включительно. В свою очередь `ushort` хранит значения от 0 до 65535.<br>
Это позволяет например исключить получение отрицательного числа в программе, или расширить диапозон положительных чисел, если не используются отрицательные.<br>

Однако, типы данных существуют не просто так (Я рассказывал почему в прошлой теме), и один из их минусов, это невозможность оперировать ими сложнее деления без конвертации в один тип.
## Конвертация данных
Конвертация происходит благодаря классу `Convert`, но так как мы пока-что не знаем что такое класс, будем считать это приставкой.<br>
Конвертировать мы можем всё что угодно во всё что угодно. Давайте для примера переведём `byte` в `ushort`.
```csharp
byte b = 69;
ushort u = Convert.ToUInt16(b);
Console.Write(u); //Выведет 69
```
Чтобы примерно понимать какой метод нужно использовать, можно представить такой алгоритм.<br>
```
<тип + имя> = Convert.To<тип данных>(<переменная>);
```
Так же, как и с `Console.ReadLine()` мы можем вставить переменную прямо в функцию вывода
```csharp
byte b = 69;
Console.WriteLine(Convert.ToUInt16(b)); //Выведет так же 69
```
Позже, когда мы затронем тему методов, мы поймём почему так можно.<br>
Но пока-что у вас скорее всего диссонанс, почему `ushort` получается путём функции `ToUInt16()`?<br>
Если кратко, т.к. `int` самый популярный тип, его решили и использовать. Цифры же обозначают битовку типа (умножаем вес типа в байтах на 8 для перевода в биты), для U-типа добавляем в начало U<br>
Таким образом для перевода в `uint` нужна функция Convert.`ToUInt32()`, для перевода в `byte` функция Convert.`ToInt8()`<br>
Но тут нету других типой по типу `float`, `string` и т.д.
Для них `Convert.` тоже есть, но спецэфичный<br>
Для:
1. `float` - Convert.`ToSingle()` (Отсюда и тип данных `double`)
2. `double` - Convert.`ToDouble()`
3. `char` - Convert.`ToChar()`
4. `string` - Convert.`ToString()`

Перевести в `var` нельзя, т.к. это подстраивающийся тип данных (Если его вообще можно назвать типом)<br>
Зная это всё, вспомним про `Console.ReadLine`
## Расскрываем суть Console.ReadLine()
Данная функция возвращает нам `string`, а если нам нужна цифра?
Ничего не понятно, щас увидите.
```csharp
//Принимаем ввод от пользователя и прибавляем к числу 1
int number = Console.ReadLine();
number += 1; //Прибавляем 1
Console.WriteLine(number); //Ещё на первой строке случится ошибка, так как ввод юзера-это строка, а мы записываем в число
```
Тут мы и используем `Convert.`
```csharp
int number = Convert.ToInt16(Console.ReadLine());
number += 1;
Console.WriteLine(number); //Теперь всё будет работать как надо
```
Сейчас всё работает, но если пользователь введёт например символ, программа ляжет, с этим мы ничего (пока-что) не поделаем.<br>
Уже со всем этим можно что-то да сделать, например, самую простейшую шифровку.<br>
